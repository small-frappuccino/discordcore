# Product Direction Checklist

Use this before adding a new feature, dashboard page, or workflow.

1. What is the slash command, and what is the workflow inside Discord?
2. Should this page exist only because it does better than chat at setup, review, bulk change, diagnostics, or recovery?
3. Does the standard UI show only current state, blockers, and the next administrative action, with everything else removed, compressed, or pushed into diagnostics?
4. If this page has many routine action buttons, is it competing with the bot instead of complementing it?
5. In one sentence, which command does this screen complement, or which real limitation of chat does it solve?
